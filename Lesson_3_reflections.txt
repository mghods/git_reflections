Q: When would you want to use a remote repository rather than keeping all your work local?
A: When you are callobrtating with others on a project or when you want to access your repo from different computers.

Q: Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
A:Other people might been working on a bug or feature or just updating the code constantly
and you don't want to keep changing your code according to their progress continuesly.So you don't want git pull it.

Q: Describe the differences between forks, clones, and branches. When would you use one instead of another?
A: When you want to copy somebody's else work and start working on it on your own is a good option (it is like copying).
When you want to copy a project from web which you are working on or particpating in to local repo, clone is the option.
When you want to pursue a new path on something you already have access to its repo and don't want to change others work
branch is the option.

Q:What is the benefit of having a copy of the last known state of the remote stored locally?
A: You can access last known state if you are online and try to modify things without messing up something good.
